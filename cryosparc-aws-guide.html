CryoSPARC on AWS

CryoSPARC 입문자를 위한 AWS 클라우드 GPU 환경 구축 가이드

AWS EC2 (Seoul) CryoSPARC v4.7.1 Amazon Linux 2 Standalone · 단일 GPU

이 가이드는 CryoSPARC을 처음 사용해 보거나 소규모 데이터로 테스트하는 경우를 기준으로, 단일 GPU(A10G) Standalone 설치를 다룹니다. 작성 환경은 AWS Seoul 리전 · Amazon Linux 2 · CryoSPARC v4.7.1이며, 시점에 따라 UI나 버전이 다를 수 있습니다. 내용이 100% 정확하지 않을 수 있으므로 AWS 공식 문서 및 CryoSPARC 공식 가이드를 함께 참고하세요. 이 가이드는 계속 수정·보완 중입니다. 대규모 처리가 필요한 경우 확장 아키텍처 섹션을 참고하세요.

AWS 인프라 설정

2-1. SSH 키 만들기

서버 접속에 필요한 "열쇠" 파일을 생성합니다.

bash 
# 키 이름(my-key-pair)과 저장 경로(~/.ssh/)는 원하는 대로 변경 가능
aws ec2 create-key-pair \
  --key-name my-key-pair \
  --region ap-northeast-2 \
  --query 'KeyMaterial' \
  --output text > ~/.ssh/my-key-pair.pem

# 보안 권한 설정 (필수!)
chmod 400 ~/.ssh/my-key-pair.pem

# Mac에서 "Operation not permitted" 시 추가 실행
xattr -c ~/.ssh/my-key-pair.pem

2-2. 보안 그룹 만들기

서버에 누가 접속할 수 있는지를 정합니다.

bash 
# 보안 그룹 생성 → 출력의 GroupId (sg-xxx) 메모!
aws ec2 create-security-group \
  --group-name cryosparc-sg \
  --description "CryoSPARC security group" \
  --region ap-northeast-2
bash — sg-XXXXXXXXXX를 위의 GroupId로 교체 
# 내 IP에서 SSH(22) 접속 허용
MY_IP=$(curl -s ifconfig.me)
aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXXXX \
  --protocol tcp --port 22 \
  --cidr ${MY_IP}/32 \
  --region ap-northeast-2

# (선택) CryoSPARC 웹 포트 직접 접속 — SSH 터널 사용 시 생략 가능
aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXXXX \
  --protocol tcp --port 39000-39010 \
  --cidr ${MY_IP}/32 \
  --region ap-northeast-2

SSH 터널을 사용하면 (Step 6) 39000~39010 포트를 열 필요가 없습니다. SSH(22)만으로 충분합니다.
네트워크가 바뀌면 (카페, 연구실 이동 등) IP가 달라집니다. 접속이 안 되면 위 명령을 다시 실행해 현재 IP를 추가하세요.

On-Demand vs Spot: 요금제 선택

EC2 인스턴스는 같은 하드웨어라도 요금 방식에 따라 가격과 동작이 크게 달라집니다.

💳 On-Demand (정가)
• 정가 요금 — 사용한 시간만큼 고정 가격
• stop/start 가능 — 서버를 "일시 중지"할 수 있음
• 강제 종료 없음 — 내가 끄기 전까지 유지
• EBS 데이터 유지 — stop해도 디스크 안전
• 가격 예시: g5.xlarge ~$1.24/hr
⚡ Spot (할인가)
• 최대 ~70% 할인
• Stop하면 데이터 삭제 — EBS 포함 모든 데이터 삭제됨
• AWS가 회수 가능 — 2분 전 경고 후 종료
• EBS는 안전 — DeleteOnTermination: false 설정 시
• 가격 예시: g5.xlarge ~$0.36/hr
On-DemandSpot
가격정가 (g5.xlarge ~$1.24/hr)~70% 할인 (~$0.36/hr)
Stop (중지) ✅ 자유롭게 Stop/Start
→ EC2 비용 0원, EBS 데이터 유지, 재시작 가능		 ⚠️ Stop하면 모든 데이터 삭제
→ Stop은 가능하지만 EBS 포함 모든 데이터가 삭제됩니다
Terminate (종료) 인스턴스 삭제
→ EBS는 설정에 따라 유지		 인스턴스 삭제 (주 사용 방법)
→ 작업 완료 후 Terminate
AWS 회수 없음 — 내가 끄기 전까지 100% 유지 2분 경고 후 Terminate 가능
EBS 데이터stop/terminate 모두 유지*terminate 시 유지*

* DeleteOnTermination: false 설정 시. 이 가이드의 서버 생성 명령에 이미 포함되어 있습니다.

어떤 걸 선택할까?
• 학습/연습 목적으로 지속 사용 → On-Demand 권장. 작업 후 Stop으로 서버를 보존하고, 다음에 Start로 이어서 사용
• 단기 집중 사용 (2~3일 프로세싱) → Spot 고려. ~70% 절감되지만 AWS가 2분 전 경고 후 회수할 수 있음

bash

4-2. 서버 상태 확인

bash — 서버에서 실행 
free -h          # RAM 확인
nvidia-smi       # GPU 확인 — "NVIDIA A10G" 보여야 정상
💡
가장 확실한 해결법: 서버 생성 시 AMI를 Deep Learning AMI GPU (Amazon Linux 2)로 선택하면 NVIDIA 드라이버, CUDA, cuDNN이 사전 설치되어 있습니다. Step 3의 AMI 검색 명령 대신 AWS 콘솔에서 "Deep Learning AMI GPU" 를 검색하세요.

Deep Learning AMI를 사용할 수 없는 경우, NVIDIA 공식 .run 설치 파일로 수동 설치할 수 있습니다:

bash — NVIDIA 드라이버 수동 설치 (Amazon Linux 2) 
# ① 빌드 도구 및 커널 헤더 설치
sudo yum install -y gcc make dkms kernel-devel-$(uname -r) kernel-headers-$(uname -r)

# ② NVIDIA 드라이버 다운로드 (버전은 최신으로 변경)
# https://www.nvidia.com/drivers 에서 Linux x86_64 → Data Center 최신 버전 확인
wget https://us.download.nvidia.com/tesla/535.183.06/NVIDIA-Linux-x86_64-535.183.06.run

# ③ 설치 실행
sudo bash NVIDIA-Linux-x86_64-535.183.06.run --silent --dkms

# ④ 재부팅 후 확인
sudo reboot
# SSH 재접속 → nvidia-smi 확인
⚠️
Amazon Linux 2의 NVIDIA 공식 yum 리포지토리 지원이 제한적이므로, .run 파일 설치가 더 안정적입니다. 드라이버 버전은 nvidia.com/drivers에서 "Data Center / Tesla" → Linux 64-bit 최신 R535 이상을 선택하세요.

4-3. 고속 SSD 마운트 (/scratch)

CryoSPARC의 임시 파일을 저장할 고속 NVMe SSD를 준비합니다.

bash — 서버에서 실행 
lsblk                                    # nvme1n1 디바이스 확인 (g5.xlarge: ~250GB)
sudo mkfs.xfs /dev/nvme1n1               # 포맷
sudo mkdir -p /scratch                   # 마운트 디렉토리
sudo mount /dev/nvme1n1 /scratch         # 마운트
sudo chown -R ec2-user:ec2-user /scratch # 권한 설정
df -h /scratch                           # 확인

CryoSPARC 설치

5-1. 다운로드

bash — 서버에서 실행 ([LICENSE_ID] 교체) 
cd ~

# Master 다운로드 (~630MB)
curl -L https://get.cryosparc.com/download/master-latest/[LICENSE_ID] \
  -o cryosparc_master.tar.gz

# Worker 다운로드 (~4GB)
curl -L https://get.cryosparc.com/download/worker-latest/[LICENSE_ID] \
  -o cryosparc_worker.tar.gz

# 압축 해제
tar -xf cryosparc_master.tar.gz
tar -xf cryosparc_worker.tar.gz

5-2. Standalone 설치

bash — 각 값을 실제 정보로 교체 
cd ~/cryosparc_master

./install.sh \
  --standalone \
  --license [LICENSE_ID] \
  --worker_path /home/ec2-user/cryosparc_worker \
  --ssdpath /scratch \
  --initial_email "your@email.com" \
  --initial_password "YourPassword#2026" \
  --initial_username "yourusername" \
  --initial_firstname "First" \
  --initial_lastname "Last" \
  --port 39000 \
  --yes
옵션설명
--standaloneMaster + Worker를 한 서버에 설치
--ssdpath /scratch임시 파일용 고속 SSD 경로
--initial_email / password웹 로그인에 사용할 계정 정보
--port 39000웹 인터페이스 포트

설치에 10~20분 소요됩니다. 자동으로 진행되니 기다리세요.

CryoSPARC는 EC2 로컬 디스크에 있는 데이터만 읽을 수 있습니다. 로컬 컴퓨터의 micrograph 파일을 서버로 전송합니다.

6-1. 데이터 저장 위치

데이터는 EBS 볼륨에 저장합니다. Stop 후에도 유지되므로 안전합니다.

mkdir -p /home/ec2-user/data

전송한 데이터를 /home/ec2-user/data/ 아래에 정리하세요. 이후 CryoSPARC에서 Import할 때 이 경로를 사용합니다.

6-2. scp로 전송 (소규모 데이터)

수십 GB 이하의 데이터는 scp로 직접 전송하는 것이 간편합니다.

bash — Mac 터미널 
# 서버에 데이터 디렉토리 생성 (SSH 접속 후)
mkdir -p /home/ec2-user/data/project01

# Mac에서 micrograph 폴더 전송
scp -i ~/.ssh/my-key-pair.pem -r \
  /path/to/micrographs/ \
  ec2-user@[PUBLIC_IP]:/home/ec2-user/data/project01/

# 전송 확인
ssh -i ~/.ssh/my-key-pair.pem ec2-user@[PUBLIC_IP] \
  "ls -lh /home/ec2-user/data/project01/ | head"

6-3. S3 경유 전송 (대규모 데이터)

수백 GB 이상의 데이터는 S3를 경유하면 안정적이고, 중단 후 이어서 전송할 수 있습니다.

bash — Mac 터미널 
# ① S3 버킷 생성 (최초 1회)
aws s3 mb s3://cryosparc-data-[계정ID]-apne2 \
  --region ap-northeast-2

# ② 로컬 → S3 업로드
aws s3 sync /path/to/micrographs/ \
  s3://cryosparc-data-[계정ID]-apne2/project01/ \
  --region ap-northeast-2

# ③ S3 → EC2 다운로드 (서버 SSH 접속 후)
aws s3 sync s3://cryosparc-data-[계정ID]-apne2/project01/ \
  /home/ec2-user/data/project01/

같은 리전(Seoul) 안에서 S3↔EC2 전송은 무료입니다. 전송이 중단되면 같은 aws s3 sync 명령을 다시 실행하면 이어서 전송됩니다.

AWS PCS 아키텍처 개요

AWS Parallel Computing Service(PCS)는 관리형 HPC 클러스터 서비스입니다. CryoSPARC과 결합하면 작업 종류별로 다른 인스턴스를 자동으로 할당할 수 있습니다.

PCS 3-Queue 아키텍처 (AWS 공식 권장)
cpu-queue
c5a.8xlarge
Import, Motion Correction
single-gpu-queue
g6.4xlarge
CTF, 2D Class, Refinement
multi-gpu-queue
g6.48xlarge
3D Refinement, Polishing
CryoSPARC의 Lane = PCS의 Queue. 작업 제출 시 적절한 Lane을 선택하면 해당 Queue의 노드가 자동 생성됩니다.
