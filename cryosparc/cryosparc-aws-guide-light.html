CryoSPARC on AWS

CryoSPARC 입문자를 위한 AWS 클라우드 GPU 환경 구축 가이드

AWS EC2 (Seoul) CryoSPARC v4.7.1 Amazon Linux 2 Standalone · 단일 GPU
⚠️
안내  이 가이드는 특정 환경(AWS Seoul 리전, Amazon Linux 2, CryoSPARC v4.7.1)에서 작성되었습니다. AWS 콘솔 UI, 인스턴스 가용 여부, 소프트웨어 버전 등은 시점과 리전에 따라 다를 수 있으므로, 실제 적용 시 본인의 환경에 맞게 확인하시기 바랍니다.
📋
이 가이드의 대상과 범위
이 가이드는 CryoSPARC을 처음 사용해 보거나, 소규모 데이터로 기능을 테스트하는 경우를 기준으로 작성되었습니다. 단일 GPU(A10G) 인스턴스에서의 Standalone 설치를 다루며, 이 구성으로도 기본적인 SPA 워크플로우(Motion Correction → CTF → Picking → 2D/3D → Refinement)를 충분히 수행할 수 있습니다.

대규모 데이터나 고급 처리가 필요한 경우:
• 수천 장 이상의 micrograph 또는 TB 단위 데이터셋 → 멀티 GPU 인스턴스 (g5.12xlarge: A10G ×4, p4d.24xlarge: A100 ×8 등) 권장
• 대규모 3D Classification, 고해상도 Refinement → 더 많은 GPU VRAM과 시스템 메모리 필요
• 여러 프로젝트 동시 처리 → CryoSPARC 클러스터 모드 (worker node 분리) 고려
• 이러한 경우의 구성은 확장 아키텍처 섹션을 참고하세요.

6본인 확인 (SMS 또는 전화)한국 전화번호 사용 가능

1-1a. IAM 사용자 설정 (보안 권장)

AWS 계정의 Root 계정(가입 시 만든 계정)은 모든 권한을 가지고 있어 위험합니다. 일상 작업에는 별도의 IAM 사용자를 만들어 사용하세요.

🔑
Root 계정 vs IAM 사용자
Root 계정 — 가입 시 이메일로 만든 계정. 모든 권한 보유. 결제 설정, 계정 삭제 같은 관리 작업에만 사용
IAM 사용자 — 필요한 권한만 부여받은 계정. EC2, S3 등 일상 작업에 사용
#단계설정값
1AWS 콘솔 → IAM → Users → "Create user"사용자 이름: cryosparc-admin
2"Attach policies directly" 선택
3아래 정책 검색 후 체크 AmazonEC2FullAccess
AmazonS3FullAccess
AmazonVPCFullAccess
4사용자 생성 완료
5해당 사용자 → "Security credentials" 탭
6"Create access key" → "CLI" 선택Access Key ID + Secret Key 메모!
🔴
Secret Access Key는 생성 시 한 번만 보여줍니다! 반드시 안전한 곳에 저장하세요. 분실 시 새로 발급해야 합니다.

1-1b. GPU Quota 증가 요청 (필수)

새 AWS 계정은 GPU 인스턴스 사용이 제한되어 있습니다. 미리 한도 증가를 요청해야 합니다.

#단계
1AWS 콘솔 → Service Quotas → Amazon EC2
2"Running On-Demand G and VT instances" 검색
3"Request increase at account level" 클릭
4새 값: 32 이상 입력 (g5.4xlarge = 16 vCPU)
5승인 대기 — 최소 24시간, 길면 수일
⏰
Quota 승인은 시간이 걸립니다. CryoSPARC 설치 전에 미리 신청하세요. Spot 인스턴스 Quota도 별도로 필요합니다: "All G and VT Spot Instance Requests"도 함께 신청하세요.

1-2. CryoSPARC 라이선스

cryosparc.com에서 학술용 무료 라이선스를 발급받습니다. 발급 시 받는 License ID를 메모해 두세요.

AWS 인프라 설정

2-1. SSH 키 만들기

서버 접속에 필요한 "열쇠" 파일을 생성합니다.

bash 
aws ec2 create-key-pair \
  --key-name cSPARC \
  --region ap-northeast-2 \
  --query 'KeyMaterial' \
  --output text > ~/Downloads/cSPARC.pem

# 보안 권한 설정 (필수!)
chmod 600 ~/Downloads/cSPARC.pem

# Mac에서 "Operation not permitted" 시 추가 실행
xattr -c ~/Downloads/cSPARC.pem

2-2. 보안 그룹 만들기

서버에 누가 접속할 수 있는지를 정합니다.

bash 
# 보안 그룹 생성 → 출력의 GroupId (sg-xxx) 메모!
aws ec2 create-security-group \
  --group-name cryosparc-sg \
  --description "CryoSPARC security group" \
  --region ap-northeast-2
bash — sg-XXXXXXXXXX를 위의 GroupId로 교체 
# 내 IP에서 SSH(22) 접속 허용
MY_IP=$(curl -s ifconfig.me)
aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXXXX \
  --protocol tcp --port 22 \
  --cidr ${MY_IP}/32 \
  --region ap-northeast-2

# (선택) CryoSPARC 웹 포트 직접 접속 — SSH 터널 사용 시 생략 가능
aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXXXX \
  --protocol tcp --port 39000-39010 \
  --cidr ${MY_IP}/32 \
  --region ap-northeast-2
💡
SSH 터널을 사용하면 (이 가이드의 Step 6 방식) 39000~39010 포트를 보안 그룹에 열 필요가 없습니다. SSH(22)만 열어도 충분하며, 보안상으로도 더 안전합니다.
⚠️
네트워크가 바뀌면 (카페, 연구실 이동 등) IP가 달라집니다. 접속이 안 되면 위 명령을 다시 실행해 현재 IP를 추가하세요.

AMI 확인 및 서버 생성

bash — AMI(운영체제 이미지) ID 확인 
aws ec2 describe-images \
  --owners amazon \
  --filters "Name=name,Values=amzn2-ami-kernel-5.10-hvm-*-x86_64-gp2" \
  --query 'Images | sort_by(@, &CreationDate) | [-1].ImageId' \
  --region ap-northeast-2
# 출력 예: ami-013c951bfeb5d9c3b → 아래에 사용
bash — AMI ID, 보안 그룹 ID를 실제 값으로 교체 
aws ec2 run-instances \
  --image-id ami-013c951bfeb5d9c3b \
  --instance-type g5.xlarge \
  --key-name cSPARC \
  --security-group-ids sg-XXXXXXXXXX \
  --block-device-mappings '[{
    "DeviceName": "/dev/xvda",
    "Ebs": {
      "VolumeSize": 500,
      "VolumeType": "gp3",
      "DeleteOnTermination": false
    }
  }]' \
  --instance-market-options '{"MarketType":"spot"}' \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=cryosparc-server}]' \
  --region ap-northeast-2

On-Demand vs Spot: 요금제 선택

EC2 인스턴스는 같은 하드웨어라도 요금 방식에 따라 가격과 동작이 크게 달라집니다.

💳 On-Demand (정가)
• 정가 요금 — 사용한 시간만큼 고정 가격
• stop/start 가능 — 서버를 "일시 중지"할 수 있음
• 강제 종료 없음 — 내가 끄기 전까지 유지
• EBS 데이터 유지 — stop해도 디스크 안전
• 가격 예시: g5.xlarge ~$1.0/hr
⚡ Spot (할인가)
• 최대 ~70% 할인 — 여유 자원을 할인 판매
• stop 불가 → terminate(삭제)만 가능
• AWS가 회수 가능 — 2분 전 경고 후 종료
• EBS는 안전 — DeleteOnTermination: false 설정 시
• 가격 예시: g5.xlarge ~$0.3/hr
On-DemandSpot
가격정가 (g5.xlarge ~$1.0/hr)~70% 할인 (~$0.3/hr)
Stop (중지) ✅ 자유롭게 Stop/Start
→ EC2 비용 0원, EBS 데이터 유지, 재시작 가능		 ⚠️ 일반적으로 Stop 불가
→ Persistent request에서만 가능
Terminate (종료) 인스턴스 삭제
→ EBS는 설정에 따라 유지		 인스턴스 삭제 (주 사용 방법)
→ 작업 완료 후 Terminate
AWS 회수 없음 — 내가 끄기 전까지 100% 유지 2분 경고 후 Terminate 가능
→ 서울 리전에서는 매우 드묾
NVMe 데이터stop 시 소멸 (재마운트 필요)terminate 시 소멸
EBS 데이터stop/terminate 모두 유지*terminate 시 유지*

* DeleteOnTermination: false 설정 시. 이 가이드의 서버 생성 명령에 이미 포함되어 있습니다.

💡
어떤 걸 선택할까?
• 처음 설치할 때 → On-Demand 권장. 설치 중 회수되면 처음부터 다시 해야 함. 작업 후 Stop으로 서버 보존 가능
• 설치 완료 후 일상 작업 → Spot 권장. 비용 절감 효과가 크고, DeleteOnTermination: false 설정으로 EBS 보존. 작업 후 Terminate
• 며칠 걸리는 대규모 처리 → On-Demand 권장. 중간에 끊기면 시간 낭비

AWS는 남는 서버 자원을 할인된 가격으로 Spot 시장에 내놓습니다. 만약 해당 자원에 대한 수요가 갑자기 늘면, AWS가 2분 전 경고와 함께 Spot 인스턴스를 회수(terminate)할 수 있습니다.

실제로 일어나나요?
서울 리전(ap-northeast-2)의 g5 인스턴스는 회수율이 매우 낮습니다. 수 개월간 운영해도 한 번도 회수되지 않는 경우가 대부분입니다. 다만 가능성은 항상 존재하므로:

• 중요한 결과는 주기적으로 EBS 또는 S3에 저장
• CryoSPARC은 작업 중단 시 자동으로 체크포인트를 저장하므로, 재시작 후 이어서 처리 가능
• DeleteOnTermination: false 설정으로 EBS 디스크는 항상 보호

서버 상태 확인

bash 
aws ec2 describe-instances \
  --filters "Name=tag:Name,Values=cryosparc-*" \
             "Name=instance-state-name,Values=running,stopped" \
  --query 'Reservations[*].Instances[*].[Tags[?Key==`Name`].Value|[0],InstanceId,State.Name,PublicIpAddress]' \
  --output table \
  --region ap-northeast-2

running 상태가 되면 PublicIpAddress가 표시됩니다. 이 IP로 접속합니다.

Deep Learning AMI를 사용할 수 없는 경우, NVIDIA 공식 .run 설치 파일로 수동 설치할 수 있습니다:

bash — NVIDIA 드라이버 수동 설치 (Amazon Linux 2) 
# ① 빌드 도구 및 커널 헤더 설치
sudo yum install -y gcc make dkms kernel-devel-$(uname -r) kernel-headers-$(uname -r)

# ② NVIDIA 드라이버 다운로드 (버전은 최신으로 변경)
# https://www.nvidia.com/drivers 에서 Linux x86_64 → Data Center 최신 버전 확인
wget https://us.download.nvidia.com/tesla/535.183.06/NVIDIA-Linux-x86_64-535.183.06.run

# ③ 설치 실행
sudo bash NVIDIA-Linux-x86_64-535.183.06.run --silent --dkms

# ④ 재부팅 후 확인
sudo reboot
# SSH 재접속 → nvidia-smi 확인

4-3. 고속 SSD 마운트 (/scratch)

CryoSPARC의 임시 파일을 저장할 고속 NVMe SSD를 준비합니다.

bash — 서버에서 실행

sudo mkdir -p /scratch                   # 마운트 디렉토리
sudo mount /dev/nvme1n1 /scratch         # 마운트
sudo chown -R ec2-user:ec2-user /scratch # 권한 설정
df -h /scratch                           # 확인
🔴
NVMe SSD는 서버를 끄면 데이터가 사라집니다!
작업 결과는 반드시 EBS(/home) 또는 S3에 저장하세요.

AWS PCS 아키텍처 개요

AWS Parallel Computing Service(PCS)는 관리형 HPC 클러스터 서비스입니다. CryoSPARC과 결합하면 작업 종류별로 다른 인스턴스를 자동으로 할당할 수 있습니다.

PCS 3-Queue 아키텍처 (AWS 공식 권장)
cpu-queue
c5a.8xlarge
Import, Motion Correction
single-gpu-queue
g6.4xlarge
CTF, 2D Class, Refinement
multi-gpu-queue
g6.48xlarge
3D Refinement, Polishing
CryoSPARC의 Lane = PCS의 Queue. 작업 제출 시 적절한 Lane을 선택하면 해당 Queue의 노드가 자동 생성됩니다.

💾 EBS 스토리지 비용

EBS(Elastic Block Store)는 EC2의 하드디스크입니다. 서버를 꺼 놓아도(Stop) 매일 과금되므로, 스토리지 관리가 비용의 핵심입니다.

EBS 용량월 비용일 환산적합한 작업

1 TB gp3$80/월~$2.7/일중간 데이터 (300~500GB raw)
2 TB gp3$160/월~$5.3/일대규모 데이터 (500GB~1TB raw)
3 TB gp3$240/월~$8.0/일MRC 형식 대용량 (890GB raw 등)
💡
스토리지 3배 법칙: CryoSPARC는 중간 결과물(extracted particles, 3D volumes 등)이 원본의 2~3배 공간을 차지합니다. 원본 데이터 크기 × 3 이상의 EBS를 확보하세요.
🚨 주의: Stop해도 과금
EC2를 Stop하면 GPU 비용은 멈추지만
EBS 비용은 계속 발생합니다.
2주 방치 시: 1TB 기준 ~$37 낭비
✅ 절약 방법
① 결과물을 S3로 백업
② EBS 볼륨 삭제 (또는 축소)
③ 재개 시 S3에서 다시 복원
ℹ️
NVMe Instance Store vs EBS: g5 인스턴스에는 고속 NVMe SSD가 포함되어 있습니다 (/scratch에 마운트). 이것은 무료이지만 서버를 Stop/Terminate하면 데이터가 모두 삭제됩니다. CryoSPARC의 SSD Cache용으로만 사용하고, 최종 결과는 반드시 EBS에 저장하세요.
