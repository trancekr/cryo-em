# /review — 멀티 관점 리뷰 스킬

## 실행 방법
- `/review 22` — 슬라이드 22 리뷰
- `/review section:overview` — overview 섹션 전체 리뷰
- `/review all` — 전체 리뷰

## 사전 로드
1. `presentation-spec.md` — 대상 슬라이드 정보
2. `design-principles.md` — 10개 섹션 디자인 규칙
3. `slide-design-reference.md` — CSS/레이아웃 기준
4. `overview.html` — 실제 슬라이드 HTML

## 4개 관점 리뷰

### 관점 1: 레퍼런스 정확성 (필수 통과)
- [ ] 모든 DOI가 slide-ref에 하이퍼링크로 포함되어 있는가
- [ ] DOI가 실존하는 논문으로 연결되는가 (research-report 참조)
- [ ] 저자/저널/연도가 DOI 메타데이터와 일치하는가
- [ ] 슬라이드 내용이 논문에서 실제로 주장하는 내용인가
- [ ] Nobel Prize 등 업적 귀속이 정확한가
- [ ] 수치 데이터(해상도, 입자 수 등)가 논문과 일치하는가
- **심각도**: 이 관점의 모든 이슈는 🔴 필수

### 관점 2: 디자인 원칙 준수

#### 2A. 시각 레이아웃 (1920×1080 기준, 반드시 스크린샷으로 확인)
- [ ] **수직 균형**: 콘텐츠가 슬라이드 중앙에 위치하는가 (위/아래 여백 대칭)
- [ ] **빈 공간 의도**: 100px 이상 빈 공간이 있으면 의도적 여백인지, 정렬 실패인지 판단
- [ ] **텍스트 줄 길이**: 한 줄 60자(영문) 이내 — 전체 폭(1920px)을 텍스트 한 줄로 채우지 말 것
- [ ] **6×6 규칙**: 불렛 6개 이내, 불렛당 6~8단어 이내
- [ ] **1슬라이드 1메시지**: 핵심 메시지가 하나인가
- [ ] **시각적 무게 중심**: 가장 중요한 요소가 가장 눈에 띄는 위치/크기인가
- [ ] **카드/패널 높이 균형**: 같은 행의 카드들이 비슷한 높이인가
- [ ] **요소 간 간격 일관성**: gap이 8px 단위로 일관적인가 (8, 16, 24, 32, 48px)

#### 2B. design-principles.md 10개 섹션
- [ ] §1 레이아웃: 히어로는 풀블리드, 카드 내부 padding 충분
- [ ] §2 배경: 바탕색 일관성, 톤 전환 부드러움
- [ ] §3 텍스트: 레퍼런스 가독성(clamp 12-16px), 제목 필요성
- [ ] §4 콘텐츠: 원본 의도 유지, 디자인이 콘텐츠 압도하지 않음
- [ ] §5 이미지: 정밀 크롭, 이미지 중복 사용 없음, 동일 크기 비교
- [ ] §6 카드: 불렛 수 균형, 컬러 코딩 일관성
- [ ] §7 CSS: 요청 속성만 변경, 글로벌 영향 없음
- [ ] §9 학술: 논문 확인 기반 설명, figure caption 참조
- [ ] §10 이미지 박스: box→arrow→box 패턴, 의미 설명

#### 2C. 수정 시 지켜야 할 것
- **한 번에 하나만 변경** — 여러 속성 동시 변경하면 뭐가 망친 건지 모름
- **수정 전후 스크린샷 비교** — 눈으로 확인 안 하고 코드만 보면 안 됨
- **flex 컨테이너 함부로 건드리지 말 것** — flex:0 0 auto vs flex:1 1 auto 차이를 이해하고 변경
- **전체 폭 패널의 텍스트**: max-width 제한 or 텍스트를 줄 나눔 (<br>) 또는 여러 줄 구성으로
- **심각도**: 🔴 필수 / 🟡 권장 / 🟢 참고

### 관점 3: CSS/HTML 품질
- [ ] 같은 컨테이너 내 단위 일관성 (px/% 혼용 없음)
- [ ] clamp() 사용 (반응형)
- [ ] 기존 CSS 클래스 활용 (불필요한 인라인 스타일 없음)
- [ ] design-locked-values 보존 여부
- [ ] 글로벌 CSS 변경 없음 (또는 영향 범위 확인됨)
- **심각도**: 🔴 필수 / 🟡 권장

### 관점 4: 콘텐츠 품질
- [ ] 영어 텍스트 문법/정확성
- [ ] 슬라이드당 1메시지 원칙
- [ ] 불렛포인트 6개 이내
- [ ] content-card ↔ img-card 간 내용 중복 없음
- [ ] 이미지 설명이 단순 묘사가 아닌 의미 연결
- [ ] 인접 슬라이드와의 내용 차별화
- **심각도**: 🟡 권장 / 🟢 참고
