# Cryo-EM 슬라이드 디자인 원칙

> Section 1 리뷰에서 도출. 섹션 2~4 제작 시 참조용. 계속 업데이트.

---

## 1. 레이아웃

- **히어로/임팩트 슬라이드**: 풀블리드 (padding:0, border-radius 없음)
- **콘텐츠 슬라이드**: 기본 패딩 유지
- **카드 내부**: 꽉 차 보이지 않도록 충분한 padding/gap 확보
- **이미지 크기**: 화면을 꽉 채우지 말고 여백 확보 (breathing room)

## 2. 배경

- **바탕색**: 현재 회색 유지 — 논문 이미지(흰 바탕)와의 일체감 고려
- **임팩트 슬라이드 배경**: 블러 + 저 opacity 이미지 or 원본 이미지 + 라디얼 그라데이션 오버레이
- **슬라이드 간 톤 전환**: 급격한 변화는 괜찮지만, opacity 조절로 부드럽게 할 수 있음

## 3. 텍스트 & 레퍼런스

- **이미지 레퍼런스**: 별도 slide-ref div 대신 이미지 위 오버레이 (bottom-right)
- **슬라이드 레퍼런스**: 교육용이므로 최소 가독성 확보 (clamp(12px,1.4vh,16px), #999)
- **슬라이드 제목**: 항상 필요한 것은 아님 (임팩트/도입 슬라이드는 제목 없이)

## 4. 콘텐츠 원칙

- **원본 슬라이드의 의도를 먼저 파악**한 후 수정 (불필요한 요소 추가 금지)
- **디자인은 콘텐츠를 지원**해야지, 압도하면 안 됨 ("너무 오버하지 말 것")
- **thought-provoking 슬라이드**: 콘텐츠 자체가 일하게, 디자인은 프레이밍만
- **빈 공간**: 적절한 이미지로 채울 수 있으면 채우되, 억지로 채우지 말 것
- **content-card와 img-card 간 중복 금지**: 위 정보 박스에 이미 있는 내용을 아래 이미지 설명에서 반복하지 말 것
- **이미지 설명은 그림 자체를 설명**: 단순 이미지 묘사가 아니라, 그 이미지가 해당 연구/개념에서 어떤 의미인지 연결

## 5. 이미지 처리

- **논문 이미지 크롭**: 필요한 부분만 정밀하게 자르기 (회색바/경계 제거)
- **저해상도 이미지**: LANCZOS 3x 업스케일 활용
- **양쪽 비교 이미지**: 반드시 동일한 크기로 맞출 것
- **이미지 위 텍스트 제거**: PIL 픽셀 조작보다 **HTML 흰색 박스 오버레이**가 간단하고 조정 가능
- **흰 배경 영역**: 이미지 뒤에 별도 div로, 전체 컨테이너가 아닌 이미지 주변만 감싸기
- **이미지 중복 사용 금지**: 같은 이미지를 다른 슬라이드에서 재사용하지 말 것 (예: s010_13 diffraction을 슬라이드 7과 8에서 동시 사용 불가)

## 6. 카드/패널 스타일

- **장비/항목명**: border-top 색상과 일치하는 컬러, font-weight:800
- **불렛 리스트 균형**: 같은 슬라이드의 카드들은 불렛 수를 맞출 것 (3개 vs 4개 X)
- **Before/After 비교**: 대각선 오버레이 배치 가능 (이미지 방향과 연결)

## 7. CSS 수정 규칙

- **요청한 속성만 변경**, 다른 속성 건드리지 말 것
- **글로벌 CSS 변경 시 주의**: 전체 슬라이드에 영향 — 변경 전 영향 범위 확인
- **최소 변경 원칙**: 간단한 문제는 간단하게 해결

## 8. 작업 방식

- **간단한 해법부터 시도**: 복잡한 방법 전에 HTML 요소 추가 같은 단순 접근 먼저
- **에디터 값 정확히 반영**: 사용자가 에디터로 보내준 좌표 → 비율 변환 → 그대로 적용
- **사용자 의도 정확히 파악**: 추측으로 여러 방향 시도하지 말고, 불확실하면 확인
- **한 번에 하나씩**: 여러 변경을 동시에 하지 말고, 하나 적용 → 확인 → 다음

## 9. 논문/학술 콘텐츠 작성 규칙

- **반드시 원본 논문 확인 후 설명 작성**: 지레짐작으로 내용을 집어넣지 말 것 — 그림 캡션, 방법론, 수치 등 모두 논문에서 직접 확인
- **그림 설명은 논문 figure caption 참조**: 논문에 있는 figure legend/caption이 가장 정확한 설명
- **Nobel Prize 등 업적 귀속 정확히**: 특정 논문이 아니라 방법론/분야 전체에 대한 수상인 경우 구분 (예: "single-particle 방법론으로 Nobel 2017" ≠ "이 논문으로 Nobel")
- **수치 데이터 정확히**: 논문에서 확인한 수치만 사용 (예: Henderson 1990 — 72 images for phases + 150 diffraction patterns for amplitudes, 3.5 Å는 in-plane only)
- **인접 슬라이드/카드 간 내용 차별화**: 같은 개념이 다른 연도에서 발전한 경우, 각각의 고유한 기여점 강조 (예: 1987 common-line vs 1994 projection matching — 둘 다 orientation 결정이지만 접근법이 다름)

## 10. 이미지 박스 (img-card) 구성 패턴

- **Key Principle 구조**: box + arrow(↓) + box 형태 — "Key principle:" 같은 라벨 없이 바로 내용
- **box+arrow 스타일**: `background:#f3eeff;border:1px solid #d4c4f0;border-radius:6px` / 화살표 `color:#7c3aed`
- **이미지 위 설명 박스**: 그림의 빈 공간에 반투명 오버레이로 배치 (z-index:2, `background:rgba(255,255,255,0.95)`)
- **이미지 라벨 (img-label)**: 이미지 옆 또는 아래에 배치, 그림이 무엇을 의미하는지 설명
- **캡션 (img-caption)**: 하단에 출처 포함 (저자명, 연도)
- **설명 언어**: 영어 (이미지 라벨, 캡션, box 내용 모두)
