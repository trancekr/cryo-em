# HTML/CSS 프레젠테이션 슬라이드 디자인 레퍼런스

> 웹 자료 조사 기반 정리. 슬라이드 제작/리뷰 시 참조용.
> 소스: reveal.js docs, Visme, HubSpot, CSS-Tricks, Smashing Magazine 등

---

## 1. 타이포그래피

### 폰트 크기 기준 (1920×1080 기준)
| 요소 | 권장 크기 | 비고 |
|------|----------|------|
| 슬라이드 제목 | 36–48px (clamp 사용) | 한눈에 읽히는 크기 |
| 본문/불렛 | 24–32px | 최소 24px 이하로 내리지 말 것 |
| 보조 텍스트 | 16–20px | 레퍼런스, 캡션 등 |
| 소형 라벨 | 12–14px | 출처 표기, footnote |

### 폰트 규칙
- **최대 2~3개 폰트** 전체 프레젠테이션에서 통일
- 하나는 제목용(bold/heavy), 하나는 본문용(regular/light)
- sans-serif 권장 (Lato, Montserrat, Open Sans, Noto Sans KR)
- **font-weight로 위계 표현**: 800(제목) → 600(강조) → 400(본문)
- line-height: 본문 1.5, 불렛 리스트 1.4~1.6

### 텍스트 양 제한
- **6×6 규칙**: 슬라이드당 최대 6줄, 줄당 최대 6단어
- 슬라이드당 총 30단어 이내 권장
- **1슬라이드 1메시지** 원칙
- 불렛포인트 남발은 구식 — 키워드 + 비주얼 조합 선호

## 2. 색상

### 팔레트 규칙
- **2~3개 주요 색상**으로 제한
- 1개 강조 색상(accent)은 핵심 정보에만 사용
- 배경-텍스트 간 **고대비** 필수 (프로젝터 환경 고려)
- 색상만으로 정보 전달 금지 → 색맹 접근성 (아이콘/텍스트 병행)

### 효과적인 색상 조합 패턴
```
① 흰배경 + 검정 텍스트 + 1색 강조 (가장 안전)
② 어두운 배경 + 밝은 텍스트 + 네온 강조
③ 단색 계열 그라데이션 (monochromatic)
```

### 의미 기반 색상 코딩
| 용도 | 색상 예시 | 비고 |
|------|----------|------|
| 주요 액션/강조 | #007ee5 (파랑) | 링크, CTA |
| 긍정/성공/After | #059669 (초록) | 개선, 완료 |
| 경고/주의 | #ff8b03 (주황) | 주의 사항 |
| 부정/Before | #e11d48 (빨강) | 문제점, 이전 상태 |

## 3. 레이아웃 패턴

### 기본 원칙
- **여백(whitespace)은 자신감의 표현** — 빈 공간을 채우려 하지 말 것
- 모든 요소는 **정렬(alignment)** 기준선 맞출 것
- 8px 단위 간격 체계 (8, 16, 24, 32, 48px)
- 슬라이드 간 **레이아웃 변화** 주어 시각적 단조로움 방지

### 자주 쓰는 레이아웃 유형

#### ① 타이틀 + 풀이미지 (Hero/Impact)
```
┌─────────────────────────┐
│                         │
│    [큰 이미지/배경]      │
│                         │
│      제목 텍스트         │
│                         │
└─────────────────────────┘
```
- padding: 0, 이미지 풀블리드
- 텍스트는 이미지 위 오버레이 (반투명 배경)

#### ② 좌우 분할 (Split)
```
┌────────────┬────────────┐
│            │            │
│   이미지   │   텍스트    │
│            │            │
└────────────┴────────────┘
```
- flex: `display:flex; gap:2%`
- 비율: 50/50 또는 60/40

#### ③ 카드 그리드 (Grid)
```
┌───────┬───────┬───────┐
│ Card  │ Card  │ Card  │
│  1    │  2    │  3    │
└───────┴───────┴───────┘
```
- `display:grid; grid-template-columns: repeat(3, 1fr); gap:2%`
- 카드 간 내용량 균형 맞출 것

#### ④ 비교 (Before/After)
```
┌────────────┬────────────┐
│  Before    │  After     │
│  (빨강)    │  (초록)     │
└────────────┴────────────┘
```
- 또는 대각선 오버레이 방식
- 컬러 코딩으로 즉시 구분

#### ⑤ 중앙 집중 (Centered)
```
┌─────────────────────────┐
│                         │
│     [핵심 이미지/도표]   │
│                         │
│      설명 텍스트         │
└─────────────────────────┘
```

## 4. 이미지 활용

### 규칙
- **모든 슬라이드에 시각 요소** 포함 (사진, 아이콘, 차트, 다이어그램)
- 고해상도 이미지만 사용 (저해상도 → LANCZOS 3x 업스케일)
- `object-fit: contain` — 비율 유지, 잘리지 않게
- `max-width: 100%` — 반응형 크기 제한
- border-radius: 8~12px로 부드러운 엣지

### 이미지-텍스트 통합
- 이미지 위 텍스트: **backdrop-filter:blur() + 반투명 배경** 활용
- 캡션/출처: 이미지 하단 또는 우하단에 작은 폰트
- 논문 이미지의 불필요한 텍스트: **HTML div 오버레이(흰박스)**로 가리기

### 이미지 배치 비율
| 용도 | 권장 비율 | 비고 |
|------|----------|------|
| 풀블리드 | 16:9 (1920×1080) | 슬라이드 전체 |
| 메인 이미지 | 화면의 50~70% | 나머지는 텍스트/여백 |
| 보조 이미지 | 화면의 30~40% | 텍스트 옆 배치 |
| 아이콘/로고 | 48~64px | 장식적 요소 |

## 5. CSS 테크닉 (HTML 슬라이드용)

### 반응형 폰트 (clamp)
```css
font-size: clamp(최소값, 선호값, 최대값);
/* 예: clamp(16px, 2.5vh, 28px) */
```

### 센터링 (Flexbox)
```css
/* 완벽한 중앙 정렬 */
.parent {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 또는 margin auto */
.child { margin: auto; }
```

### 절대 위치 (오버레이 요소)
```css
.overlay {
  position: absolute;
  left: 10%;    /* 컨테이너 대비 % */
  top: 15%;
  width: 25%;
}
```
- **항상 % 단위 사용** (px 아님) → 해상도 독립적

### 카드 스타일
```css
.card {
  background: rgba(255,255,255,0.94);
  backdrop-filter: blur(10px);
  border: 1.5px solid #e0e0e8;
  border-top: 3px solid var(--accent-color);
  border-radius: 10px;
  padding: clamp(14px, 1.8vh, 22px) clamp(16px, 2vw, 28px);
}
```

### 그리드 레이아웃
```css
.grid-3col {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 2%;
}
```

### 스택 오버레이 (reveal.js 패턴)
```css
.stack {
  position: relative;
}
.stack > * {
  position: absolute;
  inset: 0;
}
```

## 6. 피해야 할 실수

### 콘텐츠
- ❌ 불렛포인트 6개 초과
- ❌ 한 슬라이드에 여러 메시지
- ❌ 발표자 노트를 슬라이드에 표시
- ❌ 텍스트만 있는 슬라이드

### 디자인
- ❌ 3개 초과 폰트 혼용
- ❌ 4개 초과 색상 사용
- ❌ 불일치하는 간격/정렬
- ❌ 저해상도 이미지
- ❌ 여백 없이 빽빽하게 채우기

### CSS/HTML
- ❌ px 고정값 (→ %, vh, vw, clamp 사용)
- ❌ 인라인 스타일 남발 (→ 재사용 가능한 클래스 정의)
- ❌ 한 번에 여러 속성 동시 변경

## 7. 과학/기술 프레젠테이션 특화

### 데이터 시각화
- 숫자 나열 대신 **차트/그래프/pictogram** 사용
- 통계: radial percentage, bar chart 등으로 시각화
- 복잡한 데이터: 단계별로 나눠서 슬라이드 분리

### 논문 figure 활용
- 원본 figure에서 **필요한 패널만 크롭**
- figure legend은 별도 텍스트로 재작성 (이미지 속 텍스트는 안 읽힘)
- 출처 표기 필수 (Author et al., Journal (Year))

### 프로세스/워크플로우
- 순서도: 왼→오 또는 위→아래 방향
- 각 단계를 카드/박스로 구분
- 화살표나 연결선으로 흐름 표시
